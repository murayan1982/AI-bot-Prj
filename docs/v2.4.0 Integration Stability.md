v2.4.0 でやる候補:
1. facade 経由の provider 指定
   - create_text_chat_session(provider="openai" / "gemini" / "grok" など)
   - 既存 registry / routing との責務整理
   - 未対応 provider 指定時のエラーを明確化

2. facade 用のエラー整理
   - ValueError 直投げをどこまで許すか検討
   - FacadeConfigError / FacadeProviderError などを作るか検討
   - 外部アプリ側が例外を捕まえやすい形にする

3. minimal app integration example
   - examples/minimal_app_text_chat.py など
   - CLIではなく「アプリ側からFWを呼ぶ」雰囲気の最小例
   - demo app repo で使う呼び方の試作にもする

4. session config の公開範囲整理
   - TextChatSession に何を持たせるか
   - provider / preset / character_name / language などを参照できるようにするか
   - ただし内部RuntimeConfigを丸ごと公開しすぎない

5. README / docs更新
   - public facade の次ステップとして integration example を追加
   - main.py ready-to-run usage と framework API usage の違いをさらに明確化

v2.4.0 ではやりすぎない候補:
- voice facade
- VTS facade
- async streaming の本格化
- multi-session manager
- web server / API server化
- demo app本体の実装
- plugin APIの大改修
