# v2.2.0 Task List
# Provider Abstraction Foundation

## Goal

v2.2.0 の目的は、OpenAI provider を追加しながら、
LLM provider boundary が今後の拡張に耐えられるかを確認・整理すること。

OpenAI 対応そのものを大仕事にせず、
Gemini / Grok / OpenAI を同じ interface で扱える状態を目指す。

STT / TTS provider abstraction は大改造せず、
v2.2.0 では境界確認と将来課題の整理に留める。

---

## Policy

- OpenAI provider を追加する
- OpenAI は v2.2.0 では default provider にしない
- v3.0 までに OpenAI default 化する前提で設計する
- APIキーなしでも import / registry / builder validation が通るようにする
- 実API疎通は任意確認にする
- Gemini / Grok / OpenAI を BaseLLM interface で扱える状態にする
- 既存の Gemini / Grok route は壊さない
- STT / TTS は大改造しない
- provider 追加時にどこを触るべきかを明確にする

## v3.0 Direction

v3.0 では、外部の人が最初に触りやすい構成として、
OpenAI default 化を目指す。

想定 default:

```text
chat:
  primary: openai_fast
  fallback: gemini_fast

code:
  primary: openai_fast
  fallback: gemini_fast
```

ただし v2.2.0 では default route は変更しない。
