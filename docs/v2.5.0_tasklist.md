## Future Note - Voice-friendly output policy

- Consider adding a framework-level output policy for TTS-enabled sessions.
- The goal is to make LLM responses easier for speech synthesis to read aloud.
- This should be treated as output-quality guidance, not character personality.
- The policy should avoid unnecessary symbols, dense Markdown, tables, and excessive abbreviations.
- Code, commands, file paths, URLs, environment variable names, and proper nouns should remain unchanged when necessary.
- This should be enabled only when audio/TTS output is active.
