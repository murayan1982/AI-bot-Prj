# v2.6.0 Task List - App Integration Examples

## Goal

Make the public text facade easier to understand from an external app developer's point of view by adding focused examples for the APIs that already exist in v2.5.0.

v2.6.0 should be a small documentation / examples / smoke-check release.
It should not introduce large runtime changes.

## Current v2.5.0 state

Already available:

- `framework.create_text_chat_session()`
- `TextChatSession.ask(text)`
- `TextChatSession.ask_stream(text)`
- `TextChatSession.reset()`
- `TextChatSession.info`
- `TextChatSessionInfo`
- `FacadeError`
- `FacadeConfigError`
- `FacadeProviderError`
- provider/model override from the text facade
- `examples/public_text_chat.py`
- `examples/minimal_app_text_chat.py`
- `docs/public_facade.md`
- `docs/app_integration_contract.md`

Observed gap:

- README and docs mention `ask_stream()` and `reset()`, but there are no focused example files for them yet.
- `examples/minimal_app_text_chat.py` catches `FacadeError`, but there is no small dedicated error-handling example.
- `scripts/smoke_public_facade.py` checks that `minimal_app_text_chat.py` is importable, but it does not yet cover future example files.
- v2.5.0 has an app integration contract, but v2.6.0 can make that contract easier to learn by showing concrete app-style patterns.

## Scope

In scope:

- Add focused examples under `examples/`.
- Add import-only smoke coverage for the new examples.
- Update README and public facade docs to point to the new examples.
- Add release notes and a release checklist for v2.6.0.

Out of scope:

- Voice input support in the public facade.
- TTS output support in the public facade.
- Live2D / VTube Studio support in the public facade.
- RuntimeConfig exposure.
- STT / TTS provider abstraction.
- Voice-friendly output policy implementation.
- UI framework based demo app.
- Large changes to `core`, `runtime`, `llm`, `tts`, `stt`, or `live2d` internals.
